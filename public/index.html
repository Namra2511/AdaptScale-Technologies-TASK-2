How the Pipeline Works
1

Code Push to GitHub

Developer pushes code to the main branch. This event triggers the GitHub Actions workflow automatically.

2

Checkout & Install Dependencies

The GitHub Actions runner checks out the repository and runs npm install to install all Node.js dependencies.

3

Run Tests

Automated tests are executed via npm test. If tests fail, the pipeline stops and deployment is aborted.

4

SSH Deploy to EC2

On success, the runner SSHes into the EC2 instance, pulls the latest code, reinstalls dependencies, and restarts the app with PM2.

5

Health Check Verification

After deployment, a health check is performed against /health to confirm the app is running correctly on the server.
